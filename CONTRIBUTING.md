# Contributing to Awesome Dev Tools

We appreciate your interest in contributing to this project! Please follow these guidelines to help us keep this resource organized and valuable for everyone.

## How to Contribute

1. Check Existing Issues 
   - Look at our [Issues](../../issues) to see if your idea or fix has already been proposed or discussed.

2. Fork the Repository  
   - Click the “Fork” button on the top right of this repo page, then clone the fork to your machine.

3. Create a New Branch  
   - Avoid working directly on the `main` branch of your fork.  
   - For example: `git checkout -b feature/add-new-tool`

4. Make Your Changes  
   - Add new tools to the relevant file in `tools/`.  
   - If adding multiple new resources, group them logically or create separate commits.  
   - Ensure your additions follow the existing format and style.

5. Commit & Push  
   - Commit your changes with a clear message: `git commit -m "Add [Tool Name] to Frontend list"`  
   - Push your branch to your fork: `git push origin feature/add-new-tool`

6. Open a Pull Request  
   - Go to your fork on GitHub, click “Compare & pull request.”  
   - Provide a clear title and description of what you changed and why.  
   - Reference any related issues if applicable.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md) (if provided). We aim to maintain a welcoming environment where everyone can contribute.

## Thank You!

We’re excited to see what you’ll add! Thank you for helping us build an amazing collection of dev tools.
